# Code Of Conduct

## Our Standard

This project expects contributors to communicate with respect, precision, and good faith.

Examples of behavior that help the project:

- Discussing ideas and code with concrete technical arguments
- Assuming good intent and asking for clarification before escalating
- Accepting review feedback without personal attacks
- Sharing context, logs, and reproduction steps when reporting bugs

Examples of behavior that are not acceptable:

- Harassment, intimidation, or discriminatory language
- Personal attacks, insults, or repeated bad-faith arguments
- Publishing private information without explicit permission
- Spamming issues, pull requests, or discussions without technical value

## Scope

This Code of Conduct applies to project spaces, including issues, pull requests, discussions, and review threads.

## Enforcement

Project maintainers may edit, hide, lock, or remove content that violates this policy. Repeated or severe violations may lead to temporary or permanent bans from the project spaces.

## Reporting

For conduct issues, contact the maintainers through the security/contact channel documented in `SECURITY.md`.
