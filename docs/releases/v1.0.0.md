## AI News Open v1.0.0

AI News Open reaches `v1.0.0` with a stable workflow for turning domestic and international AI news into a publishable Chinese daily digest.

### Why this is `v1.0.0`

This release is the first version that formally locks the operator-facing contract for `v1.x`:

- new users can follow the docs and run the full pipeline locally in minutes
- default sources and source-specific extraction behavior now have regression coverage
- publication history enforces idempotency for the same stored digest and target
- config, CLI, HTTP API, export JSON, and database migration expectations are now documented as stable contract
- SQLite upgrades have a defined migration path and schema metadata
- deployment and troubleshooting docs are included in the repository

### What the project does

- Aggregates AI news from domestic and international RSS and Atom sources
- Cleans and deduplicates articles before storage
- Extracts article body text with source-specific cleanup for Chinese tech media
- Uses an OpenAI-compatible LLM to translate and summarize international stories into Chinese
- Generates Chinese AI daily digests
- Publishes digests to Telegram, Feishu, static sites, and WeChat draft and publish workflows
- Ships with FastAPI, CLI, SQLite storage, and a zero-build admin dashboard

### Highlights in `v1.0.0`

- Stable `v1.x` compatibility contract for configuration, CLI, HTTP API, export payloads, and migrations
- Export payloads now include `schema_version`
- Repository stats now expose SQLite `schema_version`
- `publish` and `run-pipeline --publish` persist digests before outbound publication
- Re-publishing the same stored digest to the same target is skipped by default
- `--force-republish` and `force_republish=true` provide an explicit bypass for intentional re-sends
- Added deployment, migration, and troubleshooting guides
- Added regression coverage for default source registry and legacy database migration

### Quick start

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip setuptools wheel
python -m pip install .
cp .env.example .env
python -m ainews run-pipeline --since-hours 48 --limit 30 --max-items 30 --export
python -m ainews serve --port 8000
```

Open the dashboard at `http://127.0.0.1:8000/`.

### Operator notes

- Private vulnerability reports are routed through GitHub Security Advisories
- If LLM credentials are missing, digest generation falls back to the built-in rule-based mode
- Repeat publishing for the same stored digest and target is skipped by default
- Review generated content before external publishing
