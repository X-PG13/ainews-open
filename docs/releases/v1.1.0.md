## AI News Open v1.1.0

AI News Open `v1.1.0` focuses on operations maturity after the first stable release. It adds source runtime controls, direct monitoring assets, stronger extraction governance, and a cleaner release path for maintainers running the stack continuously.

### What is new in `v1.1.0`

- Source cooldowns, maintenance mode, acknowledgement, snooze controls, and automatic recovery lifecycle
- Source alert history, recovery notifications, and a dashboard view for source runtime incidents
- Retry-aware extraction classification for `throttled`, `blocked`, `temporary_error`, and `permanent_error`
- Google News wrapper resolution before ingest, historical backfill commands, and stronger deduplication around canonical URLs
- Prometheus-compatible `/metrics`, a scheduled housekeeping workflow, and a Docker Compose monitoring profile with Prometheus and Grafana
- Expanded extraction fixtures and regression coverage for harder international sites such as TechCrunch, VentureBeat, Reuters, Wired, and Google News wrappers

### Operator improvements

- The admin dashboard now exposes source runtime controls and extraction operations instead of relying on logs alone
- CLI coverage now includes source acknowledgement, snooze, maintenance toggles, and runtime pruning
- Health, stats, and alerts now reflect source runtime state instead of treating all extraction problems as undifferentiated errors
- Monitoring can be booted locally with `docker compose --profile monitoring up --build`

### Why this matters

- Long-running instances can protect themselves from blocked or throttled sources instead of hammering failing sites
- Maintainers can acknowledge or suppress known incidents without losing runtime history
- Monitoring is now ready to wire into Prometheus and Grafana without building custom dashboards from scratch
- Google News no longer pollutes deduplication and downstream publication with wrapper links

### Quick start

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip setuptools wheel
python -m pip install .
cp .env.example .env
python -m ainews run-pipeline --since-hours 48 --limit 30 --max-items 30 --use-llm --persist --export
docker compose --profile monitoring up --build
```

Open:

- Dashboard: `http://127.0.0.1:8000/`
- Prometheus: `http://127.0.0.1:9090`
- Grafana: `http://127.0.0.1:3000`

### Operator notes

- The local monitoring compose profile provisions Grafana with `admin / admin`; change that before exposing it beyond localhost
- The PyPI publish workflow runs when the GitHub Release is published and trusted publishing is configured
- Repeat publication for the same stored digest and target remains idempotent by default
- Review generated content before sending it to external channels
